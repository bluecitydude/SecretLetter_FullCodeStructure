Frontend

The user interface is built with plain HTML, CSS, and JavaScript to keep the experience lightweight, fast, and easy to host on static deployment platforms like Vercel.

  • HTML for structure and page content.
  • CSS for the cyber-inspired visual design and responsive layout.
  • JavaScript for UI behavior, mode switching, form validation, and API integration.

Backend

The backend is a headless PHP API that performs encryption and decryption using OpenSSL. It is designed to expose a single API-only endpoint for secure payload handling.

  • PHP handles request processing, JSON parsing, and response generation.
  • OpenSSL provides industry-standard cryptographic operations used by the encryption engine.
  • Headless API means the backend does not render UI pages; it returns JSON responses that the frontend or external clients consume.

Deployment

This project is built to support separate deployments for frontend and backend:

  • Frontend: deployable on Vercel for static hosting and fast CDN distribution.
  • Backend: deployable on Render as a headless API service.

Architecture

The application is intentionally separated into two layers:

  • Client layer manages user interactions and sends requests to the API.
  • API layer handles encryption and decryption operations without persisting sensitive data.

The API endpoint is designed to be stateless: each request must include the mode, secret data, passphrase, and layer count.

Why this stack?

The chosen stack is ideal for secure utility projects because it is simple, widely supported, and easy to deploy:

  • Plain frontend tooling keeps the UI fast and dependency-free.
  • PHP + OpenSSL provides compatibility across most shared hosts and modern PHP platforms.
  • Split frontend/backend deployment enables a modern JAMstack-like workflow with a secure API backend.
